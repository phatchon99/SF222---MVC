# รายงานโครงงาน: ระบบจัดการข้อมูลผู้ป่วย (Patient Management System)
### รายวิชา: การพัฒนาเว็บแอปพลิเคชันตามสถาปัตยกรรม MVC
---

## 1. บทนำ (Introduction)
โครงงานนี้เป็นการพัฒนาเว็บแอปพลิเคชันสำหรับบริหารจัดการข้อมูลผู้ป่วยภายในคลินิกหรือสถานพยาบาล โดยเน้นการจัดเก็บข้อมูลที่เป็นระเบียบ การเก็บบันทึกประวัติการตรวจรักษาที่มีประสิทธิภาพ และสิทธิ์การเข้าใช้งานที่แบ่งสัดส่วนชัดเจนระหว่างผู้ดูแลระบบ (Admin) และบุคลากรทางการแพทย์ (Doctor/User)

## 2. สถาปัตยกรรมระบบ (System Architecture: MVC)
ระบบนี้ถูกพัฒนาขึ้นโดยใช้รูปแบบสถาปัตยกรรม **MVC (Model-View-Controller)** เพื่อแยกส่วนการประมวลผล ข้อมูล และส่วนติดต่อผู้ใช้งานออกจากกัน ทำให้ง่ายต่อการบำรุงรักษาและขยายระบบในอนาคต

### 2.1 Model (ส่วนข้อมูล)
- **เทคโนโลยี**: Python (SQLAlchemy ORM)
- **รายละเอียด**: ทำหน้าที่กำหนดโครงสร้าง Table ในฐานข้อมูล (SQLite) โดยใช้โมเดลคลาสในไฟล์ `models.py` 
    - **User Model**: จัดเก็บข้อมูลผู้ใช้ (ชื่อ, อีเมล, รหัสผ่านที่เข้ารหัสแล้ว, บทบาท)
    - **Patient Model**: จัดเก็บข้อมูลพื้นฐานผู้ป่วย (HN, ชื่อ, วันเกิด, กรุ๊ปเลือด ฯลฯ)
    - **Medical Record Model**: จัดเก็บข้อมูลการตรวจรักษาที่เชื่อมโยงกับผู้ป่วย

### 2.2 View (ส่วนติดต่อผู้ใช้)
- **เทคโนโลยี**: JavaScript (Next.js - App Router)
- **ดีไซน์**: Tailwind CSS & Lucide Icons
- **รายละเอียด**: ใช้การสร้างหน้าจอแบบ Component-based ในโฟลเดอร์ `frontend/src/app` เน้นความสวยงาม ทันสมัย และ Responsive (รองรับมือถือ) มีการใช้เครื่องมือภายนอกตามยุคนิยมอย่าง **Tailwind CSS** ในการตกแต่ง UI ให้ดูพรีเมียมและโหลดเร็ว

### 2.3 Controller (ส่วนควบคุมการทำงาน)
- **เทคโนโลยี**: Python (FastAPI Routers)
- **รายละเอียด**: ทำหน้าที่รับ Request จาก View มาประมวลผลผ่าน Logic ที่กำหนดไว้ในโฟลเดอร์ `backend/routers`
    - **Auth Router**: ควบคุมการสมัครสมาชิกและเข้าสู่ระบบ (JWT Authentication)
    - **Patients Router**: จัดการ CRUD ข้อมูลผู้ป่วย
    - **Records Router**: จัดการบันทึกประวัติการรักษา (มีการตรวจสอบ Role ห้าม Admin บันทึกประวัติเอง)
    - **Users Router**: ส่วนจัดการบัญชีพนักงานสำหรับ Admin

---

## 3. เทคโนโลยีและห้องสมุดที่ใช้ (Tech Stack & Libraries)

### Backend (FastAPI - Python)
- **FastAPI**: Framework หลักที่เน้นความเร็วและรองรับ ASYNC
- **SQLAlchemy**: ใช้สำหรับจัดการฐานข้อมูลผ่านรูปแบบ Object (ORM)
- **JOSE / Passlib / Bcrypt**: ใช้สำหรับการสร้าง Token JWT และการเข้ารหัสรหัสผ่านเพื่อความปลอดภัยขั้นสูง
- **Pydantic**: ใช้สำหรับ Validation ข้อมูลที่รับเข้ามาผ่าน API (Data Schemas)

### Frontend (Next.js - JavaScript)
- **Next.js 15**: Framework สำหรับ React ที่ได้รับความนิยมสูงสุด
- **Native Fetch API**: ใช้สำหรับสื่อสารกับ Backend (เปลี่ยนจาก Axios เพื่อความมาตรฐาน)
- **SweetAlert2**: ใช้สำหรับการทำ Pop-up แจ้งเตือนที่สวยงามและใช้งานง่าย
- **Recharts**: ใช้แสดงกราฟสถิติในหน้า Dashboard

---

## 4. โครงสร้าง Route และการกำหนดค่า (Routing & Configuration)

- **Config**: มีการใช้ไฟล์ `.env` สำหรับเก็บข้อมูลความลับ เช่น `JWT_SECRET` และ URL ของฐานข้อมูล
- **Frontend Routes**: แบ่งเป็นกลุ่มชัดเจน เช่น `/(auth)` สำหรับหน้าเข้าระบบ และ `/dashboard`, `/patients`, `/users` สำหรับส่วนงานหลัก
- **Backend Routes**: แบ่ง Prefix ชัดเจน เช่น `/auth`, `/patients`, `/records`, `/users` ทำให้ API เป็นระเบียบตามมาตรฐาน RESTful API

---

## 5. จุดเด่นและไอเดียที่เสริมเข้าไป (Extra Features)
1. **สิทธิ์การใช้งาน (Role-based Access Control)**: ระบบแยกสัดส่วนชัดเจน แอดมินจัดการบัญชีคนได้ แต่หมอเท่านั้นที่มีสิทธิ์บันทึกประวัติคนไข้
2. **ระบบลงนามแพทย์อัตโนมัติ**: เมื่อหมอกดบันทึกประวัติ ระบบจะดึงชื่อหมอคนนั้นมาบันทึกลงนามอัตโนมัติจาก Session ป้องกันการแอบอ้าง
3. **Dashboard สรุปผล**: มีการใช้กราฟวงกลม (Pie Chart) แสดงสัดส่วนผู้ป่วยแยกตามเพศ เพื่อให้ผู้บริหารมองเห็นภาพรวมได้ทันที

---

## 6. สรุปผลการพัฒนา
ระบบจัดการผู้ป่วยนี้ไม่ได้เป็นเพียงแค่ระบบ CRUD ทั่วไป แต่มีการประยุกต์ใช้เทคโนโลยีสมัยใหม่ทั้งฝั่ง Python และ JavaScript มาทำงานร่วมกันภายใต้อดีไซน์ที่เน้น User Experience (UX) ที่ดี และสถาปัตยกรรม MVC ที่ถูกต้องตามหลักการพัฒนาซอฟต์แวร์สากล
